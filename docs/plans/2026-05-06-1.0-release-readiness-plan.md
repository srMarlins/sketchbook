# Sketchbook 1.0 Release Readiness Plan

**Date:** 2026-05-06
**Status:** Proposed — companion to [feature-complete design (PRs D–G)](2026-05-06-feature-complete-design.md)
**Predecessors:** [post-UI plan](2026-05-05-post-ui-implementation-plan.md), [feature-complete design](2026-05-06-feature-complete-design.md)

## Goal

Bring everything *outside* the Kotlin codebase up to the bar implied by "we tag 1.0 and tell people to install it." The feature-complete design (PRs D–G) closes the in-app gaps; this plan closes the release-engineering, security-hardening, ops, and repo-hygiene gaps that fall out of the senior-engineer review and aren't on the D–G punch list.

## Scope

In:

- **Release packaging.** Conveyor signing/notarization, artifact-name reconciliation across CI/Conveyor/landing page, multi-OS CI matrix + release-dry-run PR gate.
- **Security defense-in-depth.** Hardenings the original review flagged that never made it into PRs A–G: `McpServer` cancellation, `GcsAuth` JWT escaping, releases-bucket lockdown, GH Action SHA pinning, WIF tightening, `SECURITY.md`.
- **Ops polish.** Release script idempotency, retired smoke test, recovery runbook, release-workflow hardening (concurrency, cache-control, bucket lifecycle).
- **Repo cleanup.** Retire `src-tauri/`, `web/`, `packages/`, `dist/`. Empty `kmp-compose` convention plugin. Repository-api/impl split. Stamp obsolete design docs.

Out (covered or deferred elsewhere):

- Anything in [feature-complete design (D–G)](2026-05-06-feature-complete-design.md).
- OS-keychain rotation (v1.1), GCS-gen CAS (v1.2), project dedup, audio analysis, `.als` writer / Macpath repair, conflict resolution UI — explicitly deferred upstream.

## Phasing

Four PRs, sequenced by "what blocks tagging 1.0" first, then by blast radius.

### PR-H: Release packaging — the only thing blocking the 1.0 tag

The single biggest reason we can't honestly tag 1.0 today is install friction. Three independent changes that all live in `.github/workflows/`, `conveyor.conf`, `tools/release.ps1`, and `site/index.html`.

**Conveyor signing/notarization.** Wire Apple Developer ID + Authenticode certs through Conveyor. `conveyor.conf:77-78` currently has `mac.signing-identity` and `windows.signing.cert` commented out. Two paths: (a) buy/rent the certs and pipe through GH secrets; (b) ship 1.0 unsigned with a documented Gatekeeper/SmartScreen bypass on the landing page. Pick (a) if budget allows — every alternative still leaves first-launch friction.

**Artifact-name reconciliation.** Pick one canonical name shape and propagate. Today:
- `site/index.html:377-379` links to unversioned `sketchbook.exe`, `sketchbook-mac-aarch64.zip`.
- `release.yml:120-125` uploads `output/sketchbook-*.x64.msix`, `sketchbook.exe`.
- `tools/release.ps1:72-74` writes notes pointing at `Sketchbook-x.y.z-windows-amd64.msi`, `*-macos-aarch64.dmg`.
- `conveyor.conf:64` declares Mac/Windows targets producing `.dmg`/`.msi`.

At least one of those will 404 on the next real release. Decide once (versioned `.dmg`/`.msi` per the runbook is the most defensible), update the four files together, and tag a `v0.0.1` end-to-end against the runbook. The acceptance check below catches name drift.

**CI matrix + release-dry-run PR job.** `ci.yml:13-27` runs only `./gradlew check` on `ubuntu-latest`. The last five release fixes (#54, #53, #52, #51, #45) all surfaced post-tag because the release pipeline is the test suite. Add:

1. A `mac` job and a `windows` job to `ci.yml` that run `./gradlew check :app-desktop:assemble`.
2. A `release-dryrun` job on PR that runs `conveyor make site` with a no-op signing key (or skips Mac notarization with a flag if Conveyor allows). Catches the artifact-name + Conveyor-config breakage *before* tagging.

### PR-I: Security defense-in-depth

Six small changes, batched into one PR because each is a few-line fix and they share a security-review motivation.

**`McpServer.handle` re-throw `CancellationException`.** `shared/mcp-server/src/commonMain/kotlin/com/sketchbook/mcp/McpServer.kt:56` — the existing `catch (t: Throwable)` swallows `CancellationException`. Same fix pattern as `JvmScanner`'s already-merged version: re-throw cancellation explicitly, fall through to JSON-RPC `-32603` for everything else.

**`GcsAuth` JWT via `kotlinx.serialization`.** `shared/cloud/src/jvmMain/kotlin/com/sketchbook/cloud/GcsAuth.kt:46-47, 125-127` builds the JWT header + claims with string concat plus a partial escape that misses control chars. SA file is trusted today, but escape brittleness compounds; switch to `Json.encodeToString` over a `JsonObject`.

**Releases-bucket public-list lockdown.** `tools/setup-releases-bucket.ps1:53-56` makes the bucket world-readable AND listable. An attacker can enumerate every release artifact (including staged-but-unannounced builds). Drop `storage.objects.list` from `allUsers` in the IAM bind, or front the bucket with a CDN that proxies only known paths.

**Pin GH Actions to commit SHAs.** `release.yml:53` (`hydraulic-software/conveyor/actions/build@v22.0`), `release.yml:113` (`softprops/action-gh-release@v2`), `secret-scan.yml:38` (`gitleaks/gitleaks-action@v2`). Tag refs can be moved by upstream; pin to `@<sha>` for any action that touches secrets, with a comment giving the human-readable version.

**WIF attribute-condition tightening.** `tools/setup-releases-bucket.ps1:113` filters by `assertion.repository_owner == 'srMarlins'` only; the SA principal binding (lines 117-123) is repo-pinned, which is what *actually* saves us, but the provider attribute condition reads as the boundary. Add `assertion.repository == 'srMarlins/sketchbook'` so the defense isn't solely on the SA binding.

**`SECURITY.md`.** Repo root is missing one. GitHub surfaces a "no security policy" banner on a public OSS repo. One paragraph: "report to srmarlins@gmail.com, expect 5-day acknowledgement, no bug bounty."

### PR-J: Ops polish

Six small ops/runbook fixes that don't fit on PR-H or PR-I but matter for the operate-it phase.

**`tools/release.ps1` idempotency.** Add `-DryRun`, fail fast on existing tag, handle missing signing key, scope `git log --pretty=…` to a base ref so a fresh repo doesn't dump full history into the changelog (same bug `release.yml:89-93` already fixed for CI).

**Retire `tools/smoke_real_backend.py`.** Imports `audio_web.app` from the FastAPI tree retired in #49/#50 — currently smokes against deleted code. Either replace with a Kotlin smoke (run `:app-desktop:assemble` + a headless launch) or delete and rely on the new release-dryrun job in PR-H.

**Backup/recovery runbook.** `docs/runbooks/release.md` exists; nothing else does. Author `docs/runbooks/recover-local-catalog.md`: how to rebuild the on-disk SQLite catalog from B2 manifests if it corrupts. Two pages: backup procedure (export the DB), recovery procedure (`audio rescan --from-cloud`), schema-migration runbook.

**`concurrency:` block on `release.yml`.** Two simultaneous tags race the bucket upload. Add `concurrency: { group: release, cancel-in-progress: false }`.

**`Cache-Control` split.** `release.yml:74-75` uses `max-age=300` on every uploaded object. Versioned binaries (the `.dmg`, `.msi`, `.exe` with the version in the path) should be `immutable, max-age=31536000`; only `metadata.properties` and `index.html` need 5-minute TTL.

**Releases bucket: object-versioning + 30-day lifecycle.** `setup-releases-bucket.ps1:47-51` doesn't enable either. An attacker who briefly compromises the WIF SA can overwrite past releases in place with no recovery. Enable object versioning + a lifecycle that expires non-current versions after 30 days.

### PR-K: Repo / architecture cleanup

Four changes, all "cosmetic" until you onboard a contributor.

**Retire dead trees.** `src-tauri/`, `web/`, `web/node_modules/`, `packages/`, `dist/audio_*.whl`, `_backfill_macpath.py` at repo root. None are gitignored at those paths despite #49/#50 retiring them. Move to `legacy/` with a one-line `legacy/README.md` that says "kept for git-blame; do not consume" *or* delete. Either way, drop hundreds of MB and fix `git status` for new contributors.

**Empty `kmp-compose` convention plugin.** `build-logic/src/main/kotlin/kmp-compose.gradle.kts` only re-applies `kmp-library` with a TODO. Move the duplicated `alias(libs.plugins.compose.multiplatform); alias(libs.plugins.compose.compiler)` block (currently in 7 feature `build.gradle.kts` files) into the convention plugin. Pure deduplication, zero behavior change.

**`:shared:repository-api` / `-impl` split (preparation for Android target).** `SqlProjectRepository` lives in `commonMain` but transitively requires the JVM-only SQLDelight driver; today this works because there's only one target. Split: `:shared:repository-api` (interfaces only, commonMain), `:shared:repository-impl` (jvmMain). Doesn't change runtime; unblocks an Android target landing without a refactor.

**Stamp obsolete design docs.** `docs/plans/2026-05-04-tauri-bundling-design.md`, `2026-05-04-tauri-native-design.md`, `2026-05-04-web-stationery.md`, `2026-05-04-home-shelves-design.md`, `2026-05-04-audio-catalog-v0.1-plan.md` are superseded by current designs but un-stamped. Add `Status: Superseded by <doc>` headers, or delete. Stops new contributors reading them as authoritative.

## Risks & non-risks

- **Signing certs cost real money and have a procurement cycle.** Apple Developer ID is $99/year + onboarding; Authenticode is harder. PR-H may need to ship its CI/artifact pieces *first* and the signing piece on a follow-up depending on the user's purchase timeline. Document the unsigned-launch bypass on the landing page in the meantime.
- **Retiring `src-tauri/` and `web/` will break any URL the user ever shared internally** that pointed at those paths. Mitigation: use `git mv` to `legacy/` rather than `rm` so the blame trail survives, and keep one redirect commit on the GH Pages site.
- **PR-J's bucket-versioning change is a one-way door** for the cost line — versioned objects accumulate storage charges proportional to release frequency until the lifecycle policy ages them out. At 1 release/week × ~200 MB/release × 30 days that's ~6 GB held; the bucket's already public, so the marginal cost is rounding error.
- **Non-risk: the security PR (I) is six independent changes.** None gate any other PR; if signing in PR-H slips, PR-I can still ship.

## Acceptance

After PR-K merges:

1. `git clone` of a fresh checkout is < 50 MB and shows zero untracked files in `git status`.
2. CI runs Mac, Windows, and Ubuntu jobs on every PR; tagging a release is a no-op against a freshly-passing CI.
3. Release-dry-run on PR catches an artifact-name change before tagging (manually break a name, observe red CI, fix, observe green).
4. Both installers (Mac `.dmg` + Windows `.msi`) launch from a click on the landing page with no Gatekeeper/SmartScreen warnings — OR the landing page documents the bypass with a screenshot.
5. `gh secret list` shows only minimum-scope secrets; every action in `release.yml` is pinned to a SHA.
6. `docs/runbooks/` contains both a release runbook and a recover-local-catalog runbook; both are dated within the last 30 days.
7. Bucket lifecycle policy active; `gsutil ls -L gs://sketchbook-releases | grep "Versioning enabled: Enabled"` returns true.
8. `gh search code "tauri"` and `gh search code "audio_web"` in this repo return only `legacy/` paths or zero hits.
